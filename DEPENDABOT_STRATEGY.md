# Elite Dependabot Workflow Strategy

This document outlines the automated dependency management strategy for **HealthBridge AI**.

## 1. Configuration Overview (`.github/dependabot.yml`)

We have implemented a **Grouped Update Strategy** to minimize noise and consolidate updates.

- **Schedule**: Weekly (Monday mornings)
- **Grouping**: All regular dependencies are grouped into single PRs per directory (Root & Frontend).
- **Limits**: Increased open PR limit to 10 to ensure nothing is missed.
- **Labels**: Automatically tags PRs with `dependencies`, `javascript`, and scope (`frontend`).

## 4. How to Handle the Current Situation

> **Context**: You currently have a PR for Next.js 16.1.5, but you verified you are locally on **16.1.6**.

**Action Plan:**
1.  Go to the Dependabot PR on GitHub.
2.  Comment: `@dependabot close` (or just close it manually).
3.  Reason: "Superseded by version 16.1.6".
4.  Push your local changes (which include the 16.1.6 lockfile) to `main`.
5.  Dependabot will re-scan and see you are up to date.
